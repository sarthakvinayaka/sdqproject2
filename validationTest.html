Validation Tests

Test Procedure

Once development is complete, we will set up a timetable for validating testing to guarantee that we are checking out the final, deliverable portal. Validation testing will be used to ensure that all of the system's components are communicating effectively with one another. As soon as we get access to all of the necessary resources, we will experiment with a few things that we couldn't try out beforehand.

In addition to functionality, the product will be evaluated based on non-functional factors such as performance, load-bearing qualities, and safety. The following types of testing will be utilized in the course of our validation checks:
  • Functional Testing
  • Load Testing
  • Performance Testing

Performance Testing

To ensure that the system is operating as expected, we are going to put it through performance testing.

Test Case # Test Case Description Expected Results
PTC-1 Scalability The portal must be able to handle additional vendors and increased traffic.
PTC-2 Speed The response time must be below the specified values mentioned while designing the website. It should ideally be below 0.2 secs, which is perceived as ‘instantaneous’ by the human brain.
PTC-3 Stability. The portal must be stable, i.e; it should be able to function over time without failure.
PTC-4 Integration The portal must work properly with regular integrations.

Load Testing

Through the use of load testing, we will ensure that the system acts as it was designed to do when subjected to stress and that it will gracefully shut down if it is unable to meet the demand.

Test Case # Test Case Description Expected Results
LTC-1 Maximum Operating Capacity It should be able to operate beyond its capacity without failure
LTC-2 infrastructure. The current infrastructure should be sufficient to run the application
LTC-3 sustainability The portal must be sustainable at peak user load.
LTC-4 Concurrent Users The system should support the maximum number of concurrent users as defined in the design document.
